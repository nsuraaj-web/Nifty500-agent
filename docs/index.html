1. Overview

The Stock Intelligence Platform is a comprehensive system designed to ingest, process, analyse, and provide insights on stock market data, with a focus on Indian stocks from the NSE 500 universe. It integrates multiple data ingestion pipelines, advanced derived computations, rating engines, and a Retrieval-Augmented Generation (RAG) vector-store-powered LLM agent to produce deep equity research reports.

2. Technology Stack

Primary technologies and services used across the platform.

  • Programming Language: Python
  • LLM & AI: Google Gemini LLM, LangChain framework, LangChain Agents
  • Vector Store: Chroma for RAG-based internal knowledge retrieval
  • Data Backend: Supabase (PostgreSQL-based hosted backend and database)
  • Web API: FastAPI
  • Frontend/UI: Streamlit app for interactive dashboards and reports
  • Data Sources: Screener.in, Yahoo Finance, Google Finance, Google News RSS feeds
  • Config & Secrets: python-dotenv for environment variables
  • Hosting: API backend and UI hosted on Render.com; source stored in GitHub
  • Security: bcrypt for password hashing in app user management

4. Core Modules

4.1 Agent & LLM Interaction

The agent layer uses LangChain v1 to orchestrate multiple tools and the Google Gemini LLM:

  • Internal RAG retrieval from the Chroma vector store
  • Recent news summarisation using Google News RSS feeds
  • External finance site scraping (Yahoo Finance, Google Finance, Screener.in) and summarisation

These tools are combined into a single LangChain agent that can answer detailed equity research questions and generate structured reports.

4.2 Data Ingestion & Merging

Multiple ingestion scripts stage data from external sources into Supabase staging tables. Subsequent merge logic:

  • Combines Screener.in, Yahoo Finance, and other source data
  • Produces consolidated, derived, and rating tables per ticker
  • Prepares unified Python dictionaries for downstream LLM/RAG pipelines

4.3 Derived Metrics Computation

The derived metrics pipeline computes:

  • Price returns (3M, 6M, 12M, YTD)
  • Valuation ratios: P/E, forward P/E, PEG, P/S, P/B, EV/EBITDA, market-cap-to-revenue
  • Profitability and margin metrics: gross, operating, net margins, ROCE, ROE (if available)
  • Growth metrics: revenue growth, EPS growth over different periods
  • Risk metrics: beta, debt-to-equity, cash conversion cycle, debtor days
  • Cash flow metrics: free cash flow margins, operating cash flow margins

Numeric values are cleaned and clamped where necessary to avoid extreme outliers and maintain consistency.

4.4 Ratings Engine

The ratings engine scores each stock along multiple dimensions:

  • Momentum score
  • Quality score
  • Valuation score
  • Growth score
  • Financial stability score
  • Cash flow score

Each dimension has configurable bandings and weights. A composite score is then derived, which maps to an investment rating and action such as Buy, Accumulate, Hold, or Avoid.

4.5 News & Finance Data Summarisation

The platform uses dedicated tools for qualitative information:

  • Fetches recent top news for a ticker using Google News RSS
  • Summarises narrative content from Yahoo Finance, Google Finance, and Screener.in company pages
  • Uses Gemini LLM to generate concise bullet-point summaries for inclusion in reports

4.6 API & UI Layer

Two main user-facing components:

  • FastAPI backend exposing endpoints (e.g., /agent, /health) for running the AI agent.
  • Streamlit frontend providing:
    • Dashboard views for ratings and fundamentals
    • Chat-style interface to ask questions about stocks or the entire universe
    • Report generation workflows using uploaded or default templates

6. Deployment & Hosting

The platform uses Git + Render.com for deployment and environment management.

  • Version Control: Entire codebase managed in a Git repository (e.g., GitHub).
  • Backend Service (FastAPI): Deployed as a web service on Render.com. Environment variables are configured in Render, including Supabase URLs, API keys, and Gemini credentials.
  • UI Service (Streamlit): Deployed separately on Render.com, pointing to the FastAPI backend via its public URL.
  • Environment Management: Local development uses .env files loaded via python-dotenv; production uses Renderâ€™s built-in environment variable configuration.
  • Data Refresh: Production deployments can optionally run ingestion and vectorstore update scripts as background jobs or scheduled tasks, depending on requirements.

7. Usage Overview

7.1 Data Refresh Workflow

  • Run Screener.in and Yahoo Finance ingestion scripts to populate staging tables.
  • Run merge and derived metrics scripts to update consolidated and derived tables.
  • Run the ratings engine to recompute stock scores and actions.
  • Execute the RAG ingestion script to rebuild or update the Chroma vector store.

7.2 Interactive Use

  • Use the Streamlit UI to:
    • Browse ratings and fundamentals for individual tickers
    • Chat with the RAG agent about specific stocks or subsets of the universe
    • Generate long-form equity research reports from templates
  • Use the FastAPI endpoint /agent to integrate the intelligence engine into other tools.

The system combines structured fundamentals, derived analytics, ownership patterns, external commentary, and latest news into a unified equity research experience.
